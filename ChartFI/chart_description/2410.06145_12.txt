The heatmaps present Spearman correlations of mean cold start time components per minute aggregated over all functions for each of the five regions, with correlation values where p < 0.05 marked with an asterisk to indicate statistical significance. Generally, cold start time tends to be positively correlated with the number of cold starts, although the strength of this correlation varies across regions. While cold start time is most consistently tied to scheduling time and deployment dependency time, deployment code time is largely weakly related to the rest, suggesting it is not a primary driver of cold start variability in most contexts. Region 1 exhibits the strongest and most interconnected structure, where cold start time tracks very closely with scheduling time and deployment dependency time; notably, there is a strong isolated correlation between scheduling and deploying dependencies here. Furthermore, scheduling time and pod allocation time are positively correlated with the number of cold starts in Region 1, a pattern that is also particularly evident in Regions 3 and 4. Region 2 shifts to a distinct, concentrated pattern where pod allocation time is the dominant companion to cold start time, while relationships with scheduling and dependency times are much weaker, suggesting resource provisioning outweighs build or scheduling effects. Region 3 returns to a mixed driver profile, with cold start time aligning strongly with scheduling time and showing a notable link to deployment code time, while Region 4 indicates multiple contributing stages with strong connections between cold start time, scheduling, dependency times, and pod allocation. Region 5 stands out as an anomaly where the number of cold starts is slightly negatively correlated with the timing measures, contrasting with the positive relationships seen elsewhere; however, like Region 1, it maintains a strong correlation between scheduling and deploying dependencies. These variations highlight significant shifts in the primary companions of cold start time—from scheduling and dependencies in Regions 1, 4, and partly 3 to pod allocation in Region 2—and warrant a closer examination of the workload mix and scaling policies in Region 5.
