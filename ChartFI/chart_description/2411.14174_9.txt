The stacked bar charts illustrate the fraction of tests resulting in failures across various translations of C programs, comparing outcomes between greybox fuzzing tests and user-provided tests. A distinct disparity is evident between the two testing methodologies; while user-provided tests generally yield high pass rates, the greybox fuzzing tests expose significant reliability issues. Specifically, translations of the same C program all fail on a large fraction of fuzz tests, with the data showing that 37% to 100% (68% on average) of these tests exhibit a discrepancy across different translations. For example, nearly all translations fail on about the same 38% of tests on shoco and the same 50% on test, with programs like urlparser reaching a 100% failure rate in the fuzzing environment. In stark contrast, the translations all pass on a large fraction of user-provided tests, where the "no fail" category dominates the distribution for most programs, particularly shoco and test which show almost complete success. This comparison highlights that while translations may appear robust under standard user scenarios, they are prone to widespread failures when subjected to the more rigorous demands of fuzz testing.
