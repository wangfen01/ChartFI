The activity timelines for nine participants (P1–P9) illustrate the progression of code authoring from 0% to 100%, with durations normalized by each participant’s total time to highlight shifts between Data Inspection (DI) and Code Authoring (CA). Participants frequently switch between code and data contexts, often fixing coding mistakes by checking data, which results in a high frequency of context switches between profiling data (DI_1) and creating new transformations (CA_1) or between verifying results (DI_2) and CA_1. While CA_1 dominates the early stages as the primary baseline authoring mode, the process is punctuated by brief DI interruptions for diagnostic or iterative moments. Midway through the progress, more sustained mode changes emerge, such as P3’s extended period of modifying written code (CA_2) between the 25%–40% range and P4’s significant DI_1 segment around the mid-30% to mid-40% range. In the second half of the experiment, P1 encounters a relatively difficult subtask requiring him to write code (CA_1) and review the data (DI_1) iteratively to adjust his coding approach. As participants near completion, DI_2 becomes more prominent, with P2 and P4 showing large verification runs in the 70%–90% region. The interleaving of CA_2 and DI_2 is common across the timelines, indicating that participants often encountered errors and typically fixed mistakes by checking the result table. This is particularly evident in the last third of the experiment for P6, who encountered an intractable bug and had to repeatedly modify the code (CA_2) and examine the result (DI_2), whereas participants like P7 and P8 maintained a more evenly interleaved switching behavior throughout the entire authoring process.
