These SHAP summary plots illustrate the impact of various predictors on model performance across four benchmarks, categorizing inputs into features derived from data [D], architecture [A], or free generations of a model [F]. In all tasks, the number of parameters and pretraining tokens heavily influences the predictions made by the regressor, establishing model scale and training compute as the dominant drivers of performance variation. This trend is most pronounced in `arc_challenge_25-shot` and `winogrande_5-shot`, where `[D] Total Tokens (B)` and `[A] Total Parameters` exhibit the widest SHAP spreads and a clear monotonic pattern; higher values consistently push predictions positively, indicating these reasoning tasks benefit strongly from larger models trained on more data. While scale remains critical, the influence of training data composition varies significantly by domain, as the percentage of code in pretraining often influences predictions negatively for reasoning tasks like ARC but positively for `humaneval_0-shot`. In the context of Humaneval, higher code exposure and longer `[A] Sequence Length` are associated with positive impacts, consistent with code generation benefiting from domain-specific data and longer context windows. In contrast, `truthfulqa_0-shot` stands out as the least scale-driven task, characterized by smaller overall SHAP magnitudes and a notable dependency on `[F] % Generated Weblike Text`, where higher values tend to move predictions negatively. This inversion suggests that a web-like generation style correlates with worse truthfulness behavior and highlights that alignment-sensitive tasks may not respond as reliably to pure scaling of tokens and parameters as reasoning or code completion tasks do.
